FOC运行流程
通过TIM1的定时器通道4触发 ADC的注入通道，ADC的转换中断中来执行FOC状态机。
基本上与硬件相关的接口都在foc_port文件里，如果要移植的话，主要就是PWM的输出以及电流的采集这部分移植好就行。
所有的功能都是按照面向对象的思想模块化编写，在foc_app文件中 FOC_Init函数中统一初始化所有的模块。

软件对应的硬件板采用的是相电流采集方式，如果是用低端电阻采样，需要自己再实现电流采样部分代码，在Foc_Current_Refresh这个函数里面实现就可以了。
默认电流传感器是使用的20A，所以无感运行的话，电流太大会失调。

foc_app.c文件中 FOC_MOTOR_t FOC_MOTOR是具体的电机及调试好的参数，可以自行打开对应电机的宏定义来选择。
在FOC_Motor_Run函数中， case MOTOR_START:中可以自行选择接下要运行的方式，无感运行-HALL运行-编码器运行。
软件默认是以酷飞2207 1960kv电机为准，以高频注入+滑模的方式运行。

如果是无感运行的话，只要电机一致，参数不偏差太多应该都没问题。
如果是编码器或霍尔运行的话，每天电机的传感器位置不一样，需要自行校准。

简单的校准方法：
    ABZ编码器:编码器的话只需要校准0点位置。使用开环运行，等到编码器触发过Z相之后，开环自增的电角度设置为0，Uq设置0，Ud设置一个固定值，此时电机就处于0°，此时编码器计算的角度数值就是偏差，在编码器enc->PhaseShift这个参数中设置-负偏差，则编码器校准完成。

    霍尔：霍尔的校准稍微麻烦一点，因为各个电机的三个霍尔位置不一定准确，所以三个霍尔组成的6个扇区可能大小不一样。让电机在无感状态下匀速运行，求出每个扇区内的时间，就能计算出每个扇区的大小。
        计算了每个霍尔扇区的大小后，还需要和编码器一样，计算霍尔的零点偏差。但是霍尔不能静止计算，因此还是需要无感运行。手动添加霍尔结构体里的PhaseShift，让霍尔计算的电角度和无感运行的电角度重合，则霍尔标定完成。

目前正点原子的PMSM电机还比较贵，UP主后续会再挑选一款便宜的带霍尔和编码器的电机进行调试。
